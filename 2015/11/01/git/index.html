版本回退

任何已经提交到 Git 的都可以被恢复. 即便在已经删除的分支中的提交,或者用–amend 重新改写的提交,都可以被恢复. 可能失去的数据,仅限于没有提交过的,对 Git 来说它们就像从未存在过一样

1. git commit –amend

修改上一次提交信息. 也可以如下使用

这时可以新加入一些文件

2. git checkout file

放弃当前更改, file回到暂存状态

3. git reset [–hard] version

–hard表示HEAD/索引和工作区均重置

version 可以只输前几位, 可以通过git log查看

回退后要查看之前版本, 可以使用git reflog命令

4. git reset file

将单个文件回退到某版本, 此时该文件暂存状态变为该版本, 而回退前的版本则变为未暂存状态, 使用git checkout file即可.
